# 만능 파일 변환기 — 단계별 계획

목표: 여러 포맷(이미지/PDF/DOCX/HWP/PPTX 등)을 서로 변환하는 웹 도구. 쉬운 것부터 클라이언트사이드(서버비 0원)로 시작하고, 어려운 변환은 나중에 서버 배포를 배우면서 확장.

## Phase 1 — 클라이언트사이드, 바로 가능 (난이도: 쉬움)
- 이미지 ↔ PDF, PDF 합치기/분할/압축
- 라이브러리: `pdf-lib`, `pdf.js`
- 예상 소요: 1~2주 (주 5~15시간)
- → 가장 먼저 착수할 단계

## Phase 2 — 클라이언트사이드, 한계 있음 (난이도: 중간)
- DOCX → PDF
- 라이브러리: `mammoth.js`(docx→html) + `html2pdf`/`jsPDF`
- 한계: 표/복잡한 서식은 깨질 수 있음. "완벽 변환" 광고 금지, 한계 명시할 것
- 예상 소요: 1~2주

## Phase 3 — 클라이언트사이드, 리스크 있음 (난이도: 어려움 — 조사 먼저)
- HWP → PDF
- 후보 라이브러리: `hwp.js`(2020년 이후 유지보수 중단), `@ohah/hwpjs`(WASM, 신생/검증 부족), `kordoc`(Node 중심, 브라우저 미검증)
- 착수 전 액션: 실제 HWP 샘플 파일로 후보 라이브러리 파싱 테스트 1주 스파이크 → 가능 여부 판단 후 진행 여부 결정
- 가치: 한국 특화 수요 높아 차별화 포인트로는 제일 강함

## Phase 4 — 서버 필요 (지금은 비현실적, 서버 배포 배우면서 도전)
- PPTX → PDF: 클라이언트사이드 무료 옵션 거의 없음(있는 건 유료 SDK/클라우드 API). 서버에서 LibreOffice headless 같은 걸로 변환하는 게 현실적
- PDF → DOCX/PPTX: 구조 복원이 필요해 서버사이드 파싱/OCR 가능성 높음
- 학습 목표: 이 단계에서 Cloudflare Workers, 작은 VPS, Docker + LibreOffice headless 같은 서버 배포 경험 자체가 포트폴리오 가치 있음

## 수익화
- 애드센스 (트래픽 기반 — SEO 키워드: "PDF 합치기", "DOCX PDF 변환", "HWP PDF 변환 무료" 등)
- 차별점: Phase 1~3은 "파일이 서버에 업로드되지 않음"이 프라이버시 마케팅 포인트
- Freemium: 무료는 파일 크기/개수 제한, 유료는 제한 해제 + 광고 제거
- Phase 4부터는 실제 서버 처리 비용이 생기므로 유료 전환 압박이 커짐 → 구독/크레딧 모델 검토 필요 시점
