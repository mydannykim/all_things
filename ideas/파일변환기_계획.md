# 파일변환기 — 전체 로드맵

목표: 이미지/PDF/DOCX/HWP/PPTX 등을 서로 변환하는 웹 도구.
클라이언트사이드(서버비 0원)부터 시작, 어려운 변환은 서버 배포 배우면서 확장.

---

## 진행 상황

- [x] **Phase 1** — 이미지→PDF, PDF→이미지, PDF 합치기, PDF 분할
- [x] **Phase 2** — DOCX → PDF
- [ ] **Phase 3** — HWP → PDF (스파이크 중)
- [ ] **Phase 4** — PPTX → PDF (서버 필요, 나중에)

---

## Phase 1 — 완료 ✅

- 이미지(JPG, PNG) → PDF
- PDF → 이미지(PNG)
- PDF 합치기 / PDF 분할
- 라이브러리: `pdf-lib`, `pdfjs-dist`

## Phase 2 — 완료 ✅

- DOCX → PDF
- 라이브러리: `mammoth.js`(DOCX→HTML) + `html2pdf.js`(HTML→PDF)
- 한계: 표/복잡한 서식/특수 폰트 깨질 수 있음 → UI에 경고 명시

## Phase 3 — 진행 중 (스파이크)

- HWP → PDF
- 후보: `@ohah/hwpjs` (WASM, 최근 업데이트 있음)
- 착수 전 실제 HWP 파일로 파싱 테스트 → 가능 여부 확인 후 구현 결정
- 세부 기술 계획: `ideas/파일변환기_Phase2-4_계획.md`

## Phase 4 — 보류 (서버 필요)

- PPTX → PDF: LibreOffice headless + VPS/Cloudflare Workers
- 세부 기술 계획: `ideas/파일변환기_Phase2-4_계획.md`

---

## 역방향 (PDF → DOCX/HWP/PPTX)

구조 복원이 불가능해 구현 안 함. Adobe도 유료로만 제공.
