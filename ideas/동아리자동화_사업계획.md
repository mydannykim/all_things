# 동아리 회비·출석 자동화 — 1인 바이브코딩 사업 계획

작성일: 2026-06-23 | 투입 가능 시간: 주 5~15시간

## 1. 문제 정의

동아리/스터디 회장·총무가 매번 겪는 반복 불편:
- 회비 걷고 누가 냈는지 카톡+엑셀로 수동 추적
- 미납자 파악, 리마인드 발송에 시간 소모
- 출석·지각 벌금이 있는 모임은 계산도 수동

본인이 동아리 생활에서 직접 겪은 불편이라 수요 검증이 가장 빠른 영역.

## 2. 타겟

1차: 본인 학교 동아리 회장/총무 — 직접 접근 가능, 무료 베타 테스터 확보 쉬움.
확장: 타 대학 동아리, 스터디 모임, 소모임 전체.

## 3. MVP 기능 범위 (한 가지에만 집중)

- 멤버 명단 업로드 (이름 + 카톡ID/전화번호)
- 회비 청구 메시지 일괄 발송 (공유 링크 방식)
- 회장이 "입금 확인" 체크 → 미납자만 자동 리마인드
- 엑셀/구글시트 내보내기

결제 연동(PG)은 v1에서 제외. 회비는 여전히 계좌로 직접 받고, 이 툴은 "추적+리마인드"만 담당. 사업자등록·PG 심사 같은 법적/행정 복잡도를 초기에 피하기 위함.

## 4. 기술 스택

- Next.js + Supabase (인증/DB) — Cursor·Claude Code로 빠르게 생성 가능한 조합
- 메시지 발송: 카카오톡 공유 링크 기반 (알림톡 API는 사업자 인증 필요해 v2로 미룸)
- 배포: Vercel 무료 티어

## 5. 4주 일정 (주 5~15시간 기준)

| 주차 | 작업 |
|---|---|
| 1주 | 동아리 회장·총무 5~10명 인터뷰 (불편함 재확인 + 월 3천원 지불 의향 테스트), 와이어프레임 |
| 2주 | DB 스키마, 인증, 멤버 업로드, 청구 메시지 발송 기능 |
| 3주 | 입금확인/리마인드, 엑셀 내보내기, 본인 동아리에서 실사용 테스트 |
| 4주 | 버그 수정, 랜딩페이지, 학교 동아리 연합회·에브리타임 등 홍보, 유료 전환 실험 |

## 6. 수익화

- 무료: 멤버 10명까지
- 유료: 동아리 단위 월 3,000~5,000원, 학기 단위 결제 (방학 비수기 고려)
- 1차 목표: 5~10개 동아리 유료 전환 → 월 1.5~5만원. 액수는 작지만 "실제로 돈 받는 제품을 혼자 만들었다"는 포트폴리오 증거가 핵심 가치.

## 7. 리스크 및 대응

- 학생 대상 지불 의향이 낮음 → 가격을 낮게 유지, 학기 단위로 부담 분산
- 시즌성(학기 초·말 몰림) → 비수기엔 출석/일정조율 등 인접 기능 추가해 상시 수요로 전환
- 결제 직접 수취 시 사업자등록 필요 → v1은 결제 미연동으로 회피, 트래픽·유료전환 검증 후 v2에서 검토

## 8. 이번 주 검증 액션

본인 학교 동아리 회장/총무 5명에게 직접 질문:
1. "회비 걷을 때 제일 귀찮은 부분이 뭐야?"
2. "이런 추적+리마인드 툴 있으면 한 달에 3천원 낼 의향 있어?"

응답에 따라 3장 MVP 범위 재조정 후 2주차부터 코딩 시작.
