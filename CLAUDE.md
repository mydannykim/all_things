# CLAUDE.md

이 저장소(all_things)에서 작업할 때 참고할 컨텍스트.

## 누구를 위한 저장소인가

- 김성현, 컴퓨터공학과 3학년, 25세
- 목표: 바이브 코딩(AI 코딩 도구 활용)으로 작더라도 실제 수익을 내는 프로젝트를 직접 만들어보며 포트폴리오를 쌓는 것
- 학업과 병행하므로 주당 투입 가능 시간은 보통 5~15시간 수준. 이 시간 안에 끝낼 수 있는 단위로 작업을 쪼갤 것.

## 저장소 구조 컨벤션

- 새 프로젝트는 루트에 새 폴더로 추가하고, 루트 `README.md`의 프로젝트 목록 표에 한 줄 추가.
- 각 프로젝트 폴더는 자체 `README.md`를 가지며 최소한 다음을 포함: 무엇을 하는 도구인지, 로컬 실행/테스트 방법, 배포 방법, 알려진 한계.
- 아직 구현 전이거나 보류된 아이디어는 `ideas/`에 markdown으로만 남겨두고, 실제로 만들기로 정하면 그때 프로젝트 폴더로 옮김.

## 기술적 기본 방향

- 가능하면 서버 비용이 들지 않는 구조를 우선 시도: 정적 사이트 + 클라이언트사이드 처리(브라우저 내 연산), 또는 Supabase/Vercel/Cloudflare 같은 무료 티어.
- 배포는 기본적으로 Cloudflare Pages 사용 (GitHub 연동 시 push할 때마다 자동 배포됨).
- 결제/회원 기능처럼 사업자등록이나 법적 요건이 걸리는 부분은 MVP 단계에서는 최대한 피하고, 수요 검증 이후에 추가 검토.
- 광고 수익화(애드센스 등)를 고려하는 프로젝트는 개인정보처리방침/문의 페이지를 기본으로 포함시킬 것 (승인 요건).

## 작업 진행 방식

- 아이디어를 정하기 전에 "누가 이 불편함을 직접 겪는가"를 먼저 확인 — 본인이 직접 겪었거나 가까운 커뮤니티(동아리, 학교, 취준 등)에서 확인 가능한 문제를 우선.
- 큰 기능을 한 번에 만들지 말고, 검증 가능한 가장 작은 단위(단일 기능)부터 배포.
- 매 프로젝트마다 "이번 주 검증 액션" 같은 다음 행동을 README나 ideas 문서에 명시해 다음 세션에서 바로 이어갈 수 있게 할 것.
